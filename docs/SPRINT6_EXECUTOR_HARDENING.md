## üì¶ S6.1: Executor Hardening (CORE)

### ExecutorBase - Production Foundation

**Location:** `backend/app/modules/factory_executor/base.py`

**Key Features:**
- ‚úÖ Strict input/output contracts
- ‚úÖ Fail-closed validation (invalid input ‚Üí STOP)
- ‚úÖ Idempotency support (safe re-runs)
- ‚úÖ Explicit rollback hooks
- ‚úÖ Timeout enforcement (default 5min, max 1hour)
- ‚úÖ Retry limits with exponential backoff
- ‚úÖ Execution history tracking
- ‚úÖ Audit event integration (ready)

### Capabilities System

```python
class ExecutorCapability(str, Enum):
    IDEMPOTENT = "idempotent"        # Can safely re-run
    ROLLBACKABLE = "rollbackable"    # Can undo changes
    ATOMIC = "atomic"                # All-or-nothing
    RESUMABLE = "resumable"          # Can continue after failure
```

### Execution Flow

```
User Request
    ‚Üì
validate_input_strict()  # Fail-closed: any error ‚Üí STOP
    ‚Üì
check_already_executed() # If IDEMPOTENT + found ‚Üí return cached
    ‚Üì
dry_run_execute()        # If dry_run ‚Üí validate only
    ‚Üì
execute_with_timeout()   # Main execution with timeout
    ‚Üì
record_execution()       # Cache result for idempotency
    ‚Üì
Return StepResult
```

### Validation Rules

**MUST validate:**
- Required parameters present
- Parameter types correct
- Values within acceptable ranges
- Timeouts reasonable (0 < timeout ‚â§ 3600s)
- Retry count ‚â§ max_retries

**Fail-Closed Principle:**
```python
if errors:
    raise ValidationError(f"Input validation failed: {errors}")
# No implicit "best effort" - stop on any error
```

### Idempotency

Executors with `IDEMPOTENT` capability:
1. Compute execution hash (SHA256 of parameters)
2. Check execution history
3. If found + successful ‚Üí return cached result
4. If not found or failed ‚Üí execute normally

**Execution Hash:**
```python
hash_data = {
    "step_id": step.step_id,
    "executor": step.executor.value,
    "template_id": step.template_id,
    "parameters": step.parameters,
    "execution_mode": context.execution_mode.value,
}
hash = SHA256(json.dumps(hash_data, sort_keys=True))
```

### Rollback Support

Executors with `ROLLBACKABLE` capability:
- MUST implement `rollback(step, context) -> bool`
- SHOULD record rollback actions during execution
- CAN declare non-rollbackable steps explicitly

**Rollback Contract:**
```python
async def rollback(self, step: ExecutionStep, context: ExecutionContext) -> bool:
    """
    Undo changes made by this step.

    Returns:
        True if rollback succeeded
        False if rollback failed or not supported

    Raises:
        RollbackError: Critical rollback failure
    """
```

## üîí Fail-Closed Principles (Enforced)

### 1. Input Validation

**Rule:** Invalid input ‚Üí ValidationError ‚Üí STOP

```python
# Example: Missing required parameter
params = {"domain": "acme.example"}  # Missing "template" and "business_name"
# ‚Üí ValidationError: "Missing required parameters: template, business_name"
```

### 2. Unknown Executors

**Rule:** No executor registered ‚Üí ExecutionError ‚Üí STOP

```python
# Example: Typo in executor type
step.executor = "webgenn"  # Should be "webgen"
# ‚Üí ExecutionError: "No executor registered for type: webgenn"
```

### 3. Dependency Violations

**Rule:** Unmet dependency ‚Üí ValidationError ‚Üí STOP

```python
# Example: Step depends on non-existent step
step_5.depends_on = ["step_99"]
# ‚Üí ValidationError: "Step step_5 depends on non-existent step: step_99"
```

### 4. Timeout Enforcement

**Rule:** Execution exceeds timeout ‚Üí TimeoutError ‚Üí ROLLBACK

```python
# Example: Step takes too long
context.timeout_seconds = 60  # 1 minute
# If execution takes > 60s ‚Üí TimeoutError ‚Üí auto-rollback (if enabled)
```

### 5. Partial State Prevention

**Rule:** Step failure ‚Üí ROLLBACK (if auto_rollback=true) ‚Üí NO partial deployment

```python
# Without auto-rollback: Steps 1-3 completed, Step 4 failed ‚Üí PARTIAL STATE ‚ùå
# With auto-rollback: Steps 1-3 completed, Step 4 failed ‚Üí ROLLED BACK ‚úÖ
```
