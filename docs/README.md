# Documentation Index

This folder is organized into active documentation and historical archives.

## Active Documentation

- `architecture/` - System architecture, target/current state, conceptual framework
- `auth/` - Authentication and authorization tracking and knowledge base
- `deployment/` - Deployment guides and operational deployment instructions
- `frontend/` - UI layout and frontend structure documents
- `governance/` - Governance, constitutional model, and safety controls
- `guides/` - Developer and workflow guides
- `infrastructure/` - DNS and infrastructure-specific integration docs
- `knowledge/` - Knowledge layer and persistence concepts
- `modules/` - Module-specific docs (`axe`, `webgenesis`, `course_factory`, `credits`)
- `mission_system/` - Mission subsystem docs (plus internal mission archives)
- `security/` - Security posture, egress, safe mode, and lockdown docs
- `skills/` - Skill engine and capability composition docs
- `specs/` - Detailed technical specs

## Archive

- `archive/root/` - Non-core markdown files moved from repository root
- `archive/sprints/` - Historical sprint documents
- `archive/tasks/` - Historical task implementation reports
- `archive/reports/` - Historical status and implementation reports
- `archive/sessions/` - Session summaries and completion notes
- `archive/testing/` - Historical testing output and reports
- `archive/strategic/` - Legacy roadmaps and strategic planning docs
- `archive/releases/` - Historical release notes
- `archive/legacy/` - Legacy or obsolete documentation snapshots
- `archive/pr-193-credits/` - Archived PR-193 credits/event-sourcing documentation

## Root-level policy

Keep only core project markdown files in repository root:

- `README.md`
- `README.dev.md`
- `CHANGELOG.md`
- `CLAUDE.md`
- `TODO.md`
- `TODO-2026-02-23.md`
